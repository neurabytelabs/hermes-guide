Bölüm 1

Önce yanlış modeli kır

Zihinsel model yanlışsa her özellik kötü kullanılır. Hermes sohbet kutusu olarak güçlü değildir. Gerçek bir operasyon çerçevesi içinde çalıştığında değerli olur.

Yanlış model
  • İzole sorular sor
  • Her seferinde bağlamı sıfırla
  • Tek promptun her şeyi çözmesini um
  • İncelemeden önce eylem iste
  • Başarılı işi sonsuza kadar elle tekrarla
Bu makul cevaplar, tutarsız sonuçlar ve verimlilik kılığına girmiş kaos üretir.
Doğru model
  • Kalıcı bağlam ver
  • Önce mevcut durumu incele
  • Bir şeyi değiştirmeden önce planla
  • Gerçekliği doğrulamak için araçları kullan
  • İşe yarayanı yakala ve tekrarı otomatikleştir
Bu Hermes’i akıllı bir yanıtlayıcıdan operasyon sisteminin parçasına dönüştürür.

Bölüm 3

            
Hermes ile ilk 15 dakika

          

          
Pratik başla. Hermes’i aç, modeli seç, sistemi doğrula, sonra saf sohbette kalmak yerine araçlarla temellenmiş bir iş çalıştır.

        

        

          

            
01 / Kurulum

            
Sihirbazı bir kez çalıştır.

            
Provider, model, terminal, gateway, araçlar ve ajan varsayılanlarını etkileşimli kurulumla yapılandır.

            hermes setup
          

          

            
02 / Sağlık

            
Operasyondan önce kontrol et.

            
Eksik bağımlılıkları, kimlik doğrulama sorunlarını ve gateway/araç yapılandırmasını sağlık komutlarıyla yakala.

            hermes doctor
hermes status
          

          

            
03 / Model

            
Doğru motoru seç.

            
Hermes provider’dan bağımsızdır. İş akışını değiştirmeden model ve provider değiştirebilirsin.

            hermes model
          

          

            
04 / Tek seferlik

            
Script’ler için non-interactive mod kullan.

            
Otomasyon ve tekrarlanabilir kontroller için shell’den tek sorgu çalıştır ve nihai çıktıyı al.

            hermes chat -q "Bu repoyu incele ve riskleri özetle"
          

        

      

    


    

      

        

          

            
Bölüm 4

            
Güncel operasyon modeli

          

          
Hermes en çok kalıcı bağlam, araçlar, beceriler, hafıza, zamanlama ve mesajlaşma aynı operatör döngüsüne hizmet ettiğinde güçlenir.

        

        

          

            
01 / CLI

            
Terminal merkezli, otomasyona hazır.

            
Etkileşimli iş için hermes, tek seferlik uygulama için hermes chat -q, paralel ajanlar izole git alanı istediğinde --worktree kullan.

            
CLI kontrol odasıdır; sohbet yalnızca arayüzlerden biridir.

          

          

            
02 / Araç setleri

            
Cevapları gerçekliğe bağla.

            
Terminal, file, web, browser, vision, image generation, TTS, todo, cronjob, delegation, memory, session search ve skills gibi odaklı araç setlerini kullan.

            
Gerçeklik kontrol edilebiliyorsa tahminle yetinme.

          

          

            
03 / Beceriler + hafıza

            
İşe yarayanı biriktir.

            
Hafıza kalıcı gerçekleri ve tercihleri tutar. Beceriler yeniden kullanılabilir prosedürleri, komutları, tuzakları ve doğrulama adımlarını tutar.

            hermes skills search github
hermes skills install <skill-id>
hermes memory status
          

          

            
04 / Gateway

            
Zaten bulunduğun yerden operasyon yap.

            
Gateway Hermes’i Telegram, Discord, Slack, WhatsApp, Signal, Email, SMS, Matrix, Mattermost, Home Assistant, Feishu, WeCom, BlueBubbles, Weixin, API Server ve webhook’lara taşır.

            hermes gateway setup
hermes gateway status
          

          

            
05 / Cron + webhook

            
Kontrolleri tekrarlayan operatörlere dönüştür.

            
Zamanlanmış işler için cron, olay tetiklemeli işler için webhook kullan. Faydalı promptların operasyon rutinine dönüştüğü yer burasıdır.

            hermes cron create "0 9 * * *"
hermes webhook subscribe deploy-watch
          

          

            
06 / MCP + plugin

            
Ajanı baştan yazmadan genişlet.

            
İş akışı daha özel araçlara ihtiyaç duyduğunda MCP sunucuları bağla, plugin kur ve gerekirse Hermes’in kendisini MCP sunucusu olarak yayınla.

            hermes mcp list
hermes plugins list
          

        

      

    


    

      

        

          

            
Bölüm 5

            
Güvenli biriken iş akışı

          

          
İyi Hermes kullanımı şanslı sohbetler dizisi değildir. Kontrollü bir döngüdür: bağlam → incele → planla → uygula → doğrula → biriktir.

        

        

          

            
1

            
Bağlam

            
Proje, kısıtlar, doğruluk kaynakları, başarı ölçütleri ve dokunma alanları gibi kalıcı bilgileri yükle.

          

          

            
2

            
İncele

            
Bir şeye dokunmadan önce dosyaları, dokümanları, logları, web sayfalarını, issue’ları ve sistem durumunu oku.

          

          

            
3

            
Planla

            
İşi geri alınabilir adımlara böl, riskleri görünür yap ve en küçük güvenli hamleyi seç.

          

          

            
4

            
Uygula

            
Araçları kasıtlı kullan. Yıkıcı, geniş etkili veya dış dünyayı etkileyen işlemlerden önce onay iste.

          

          

            
5

            
Doğrula

            
Test, build, tarayıcı kontrolü, API yoklaması veya canlı smoke test çalıştır. Komutları ve sonuçları net raporla.

          

        

        
Sonra biriktir: yeniden kullanılabilir prosedürleri beceri olarak kaydet, kalıcı gerçekleri hafızada tut, tekrar eden kontrolleri cron ile zamanla ve acil iş akışlarını gateway üzerinden yürüt.

      

    


    

      

        

          
Bölüm 6

          
Kişisel operasyon sistemini kur.

          
Hermes sohbet botu olmayı bırakıp uygulama şeklinin parçası olduğunda güçlenir: terminal, dosyalar, tarayıcı, hafıza, beceriler, zamanlamalar, webhook’lar ve mesajlaşma tek disiplinli döngüye bağlanır.
